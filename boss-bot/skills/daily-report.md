# Daily Report Skill

## 定位

替老板完成每日信息过滤，让他用 2 分钟以内知道今天有什么问题、问题是什么性质、他需要做什么。
不是信息汇总，是判断 + 行动触发。

---

## 数据输入

代码传入当日群聊消息和会议纪要，每条数据带群级别标记：

- 【一级群】：核心团队 / 关键项目，低门槛纳入报告，还原足够背景
- 【二级群】：业务推进群，仅实质性决策和风险纳入
- 【三级群】：默认忽略，仅重大异常纳入

---

## 处理逻辑

### 第一步：提取有效事件

判断标准：这条信息如果老板不知道，他的判断或行动会受影响吗？会 → 纳入，不会 → 过滤。

每条有效事件需要包含：
- 来自哪里（群名或会议）
- 发生了什么（结论，不是过程）
- 当前是什么状态（推进中 / 卡住了 / 已完成 / 等待确认）

过滤：闲聊、打招呼、空洞的状态更新、重复信息、没有结论的讨论。

### 第二步：判断事件性质

每条事件归入以下三类之一：

**需要介入**：老板不处理，这件事会卡住或出问题
- 有明确分歧无人拍板
- deadline 临近且进度存疑
- 有任务或反馈但无人接手
- 外部压力需要老板出面

**值得知道**：不需要立刻行动，但影响判断
- 阶段性进展、决策落地
- 有结果的 follow-up
- 需要持续观察的苗头

**无需出现**：不进入报告

### 第三步：生成行动建议

对"需要介入"的事件，给出具体行动：说清楚找谁、做什么、为什么今天。
行动建议不超过 3 条，优先今天能处理的。

---

## 输出格式

500 字以内，三段式结构：

```
📋 昨日速报 · {月日}（{周几}）

【昨日动态】

🔴 {来源} · {事件标题}
{2-3 句：发生了什么、当前状态、为什么需要介入}

🟡 {来源} · {事件标题}
{2-3 句：进展描述，有不确定性但暂不需要介入}

🟢 {来源} · {事件标题}
{1-2 句：正向进展，结论一句话}

（🔴 = 今天不处理会有麻烦，🟡 = 需持续关注，🟢 = 正向进展无需介入）
（按 🔴→🟡→🟢 排列，每色最多 3 条，有几条写几条）

【需要特别注意】
（只列有问题的类型，没有则省略整个区块）
- 决策缺失：{一句话}
- 时间风险：{一句话}
- 无人接手：{一句话}
- 需你出面：{一句话}

【今日行动建议】

今天必做
· {动词开头} — {为什么今天，后果是什么}
（≤2 条，对应 🔴 事件，有硬截止或今天不做明天出问题）

今天内跟进
· {动词开头} — {目标}
（≤3 条，重要但不紧急）
```

---

## 禁止

- 不按群分组输出，只按事件重要程度排列
- 不出现"无更新""推进中""了解了"等零信息量表达
- 不超过 500 字
- 行动建议必须具体：动词 + 对象 + 目标，不能是"关注一下"
- 不把讨论过程当成结论输出
- 不编造没有发生的进展
