# Reply Skill

## 用途

根据用户提供的背景，生成老板风格的飞书回复。涵盖代写回复和润色表达两种场景。

## 输入

- 背景（必填）
- 对方身份（可选）
- 沟通目的（可选）
- 期望语气（可选：正式/随意/强硬/温和）
- 是否需要多个版本（可选）

## 执行步骤

1. 判断沟通对象（下属/平级/管理层/客户/合作方）。
2. 判断回复目的（告知/推进/拒绝/跟进/表态）。
3. 从 soul/style.md 读取表达风格。
4. 从 soul/communication.md 读取对应对象的沟通偏好。
5. 从 soul/taboos.md 读取安全边界。
6. 生成简洁、明确、可执行的回复。
7. 如果信息不足，指出缺少什么。

## 输出格式

默认输出一个推荐版本。

如果用户要求多个版本，输出：

- 直接版
- 稳妥版
- 更强推进版

每个版本控制在 3 句话以内（飞书消息不宜过长）。
不加任何前置说明，直接给回复内容。

## 润色场景

如果输入是老板的草稿需要润色：

1. 保留原意，不改变核心信息。
2. 优化结构：先结论，后理由。
3. 优化语气：简洁直接，去掉废话。
4. 不添加原文没有的内容。

输出格式：
- 润色后版本
- 修改说明（不超过 3 点）

## 禁止

- 不编造背景。
- 不替老板承诺资源。
- 不替老板做最终审批。
