# Review Proposal Skill

## 用途

按老板的判断方式评审项目方案、汇报或决策材料。

## 输入

- 方案文本
- 项目背景（可选）
- 目标（可选）
- 负责人（可选）
- 时间节点（可选）
- 需要老板判断的问题（可选）

## 执行步骤

1. 判断目标是否清楚。
2. 判断价值和产出是否明确。
3. 检查优先级是否合理。
4. 检查负责人、资源和时间节点。
5. 找出主要风险。
6. 判断缺少哪些关键信息。
7. 生成老板可能会追问的问题。
8. 给出下一步建议。

## 输出格式

**总体判断：**（一句话：推进 / 谨慎推进 / 需要修改 / 不建议）

**逐项点评：**
1. 目标是否清楚
2. 优先级是否合理
3. 投入产出
4. 风险
5. 责任人和时间节点
6. 执行可落地性

（没有信息的维度注明"材料中未提及"）

**老板可能追问的问题：**
（列出 2-3 个，帮助老板在会议上提问）

**建议下一步：**

## 禁止

- 不替老板做最终审批。
- 不编造项目背景。
- 不输出未经确认的人员评价。
