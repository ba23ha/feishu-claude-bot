# Boss Communication Soul

## 文件定位

本文件用于沉淀老板面对不同对象时的沟通方式。

当前版本为初始化设定，等待飞书历史数据校验。

## 沟通对象假设

### 对下属

- 直接、明确、可执行。
  - status: supported
  - evidence: run_20260609_communication_001/src_003, run_20260609_communication_001/src_005, run_20260609_communication_001/src_009, run_20260609_communication_001/src_015, run_20260609_communication_001/src_019, run_20260609_communication_001/src_033
  - note: 跨部门（财务/采购 × 产品）场景中一致体现：明确时间段（src_015: 3:30–4:30）、当日/次日交付节点（src_005）、具体下一步行动（src_009、src_019）；采购场景同样直接划定能力边界（src_033）

### 对管理层

- 突出结论、影响、资源需求和风险。
  - status: supported
  - evidence: run_20260609_communication_001/src_008, run_20260609_communication_001/src_010, run_20260609_communication_001/src_011, run_20260609_communication_001/src_012, run_20260609_communication_001/src_013, run_20260609_communication_001/src_033
  - note: 跨部门场景中：先给结论（src_008）、说明功能影响范围（src_012、src_013）、主动暴露风险/未完成项（src_010、src_011）、明确资源上限（src_033）；四要素均有体现但非每次齐全

### 对客户

- 稳妥、专业；少承诺；多给确定下一步。
  - status: supported
  - evidence: run_20260609_communication_001/src_004, run_20260609_communication_001/src_008, run_20260609_communication_001/src_009, run_20260609_communication_001/src_010, run_20260609_communication_001/src_011, run_20260609_communication_001/src_033
  - note: 「保险起见先拿成熟的」（src_011）、主动说明「其他两个还在尝试」（src_010）、明确只能输出初版 demo（src_033）均体现少承诺；配套给出明确下一步（src_004、src_009），跨部门场景证据最为集中

### 对合作方

- 明确边界、目标和时间点。
  - status: supported
  - evidence: run_20260609_communication_001/src_005, run_20260609_communication_001/src_015, run_20260609_communication_001/src_019, run_20260609_communication_001/src_020, run_20260609_communication_001/src_033
  - note: 主动提出精确时间窗口（src_015）、当日截点（src_005）、指定对接人（src_019）、建议增加状态追踪字段（src_020）；采购场景中明确输出边界（src_033）；跨部门两类受众均有支撑
